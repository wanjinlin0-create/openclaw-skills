# 演示任务：设计多智能体可视化平台

## 任务描述

设计并实现一个**局域网多智能体可视化交流平台**，让多个AI智能体能够协作完成任务，并通过Web界面可视化展示它们的交互过程。

## 具体需求

### 功能需求
1. **多智能体接力系统**
   - 支持4个角色的智能体：分析员、策划师、执行者、审核员
   - 任务按顺序在智能体间传递
   - 每个智能体有明确的输入输出

2. **可视化界面**
   - Web界面，局域网内可访问
   - 实时显示各智能体状态（空闲/工作中/完成）
   - 展示消息接力流程图
   - 可查看每个阶段的详细输出

3. **任务管理**
   - 可提交新任务
   - 查看历史任务列表
   - 可重新运行某一步骤

### 技术需求
- 后端：Python (FastAPI) + WebSocket
- 前端：HTML5 + JavaScript (或 Vue/React)
- 通信：WebSocket 实现实时推送
- 部署：Docker 化，一键启动

### 约束条件
- 必须在局域网内运行，无需公网
- 界面要简洁直观，非技术人员也能使用
- 支持多任务并行执行

## 成功标准
1. 能在浏览器中打开界面，看到4个智能体的状态面板
2. 提交任务后，能看到任务在各智能体间传递
3. 最终输出完整的执行结果和审核报告
4. 代码结构清晰，易于扩展新角色
